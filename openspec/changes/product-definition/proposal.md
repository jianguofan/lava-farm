# Proposal: 产品定义与产品中心

## Summary

建立农场生产用的产品/模型定义能力，把外部 G-code、Gcode.3MF、缩略图、物料颜色、物料克重、机型、生产时长、单盘数量沉淀为可复用产品库。产品中心以卡片网格展示产品，并作为批量投产流程的入口。

## Motivation

当前群控底座已经能发现设备、聚合状态、上传文件和批量下发命令，但缺少“投产对象”的稳定定义。现场使用不能每次只选择裸文件，否则无法提前校验机型、耗材颜色、单盘数量和预计时长，也无法复用历史模型。

参考版本计划中模块①“产品配置”，产品库需要支撑后续预打印配置页和批量投产，让操作员先选产品，再选设备，最后投产。

## What It Does

- 从 G-code / Gcode.3MF 导入产品定义。
- 提取或录入：产品名、版本、机型、生产时长、物料总重、单盘数量、颜色/克重、缩略图。
- 产品中心按截图式卡片展示产品。
- 支持搜索、筛选、编辑、删除、版本自增。
- 从产品卡片直接进入投产流程。

## Non-goals

- 不在阶段一实现在线切片。
- 不接入模型站下载。
- 不做云端产品库同步。
- 不处理复杂 BOM/库存扣减。

## Success Criteria

- 操作员能在产品中心看到所有产品卡片和关键生产参数。
- 导入 G-code / 3MF 后能生成可编辑产品定义。
- 产品卡片点击“投产”可携带产品进入预打印流程。
- 产品数据本地持久化，重启应用后仍可查看。
