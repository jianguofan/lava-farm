# Proposal: 异常置顶提示

## Summary

建立异常检测、合并、置顶提示和确认机制，让设备异常在控制面板顶部持续可见，并在设备卡片上突出显示。

## Motivation

版本计划模块⑤要求异常响应 ≤ 1 分钟，并提供视觉和语音提醒。当前设备状态已经进入 `FarmStore`，但尚缺少独立告警模型、告警引擎和置顶 UI。

## What It Does

- 定义异常类型和严重度。
- 从 MQTT/HTTP 状态和通知中生成告警。
- 控制面板顶部展示置顶异常横幅。
- 异常设备卡片红框/强调。
- 支持确认、解决、静音时段。

## Non-goals

- 不实现短信/云端推送。
- 不实现多用户权限。
- 阶段一语音播报可先预留接口。

## Success Criteria

- 设备异常后 1 分钟内在 UI 置顶显示。
- 操作员可确认告警。
- 同类重复告警会合并而不是刷屏。
